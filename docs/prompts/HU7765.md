# HU7765 - Agregar Criterio EUR-ACE

## Descripción
Como usuario, quiero poder agregar un nuevo Criterio EUR-ACE a través de un endpoint, para poder incluir criterios que se alineen con los estándares de calidad educativa.

## Criterios de Aceptación
1. Se debe crear un endpoint POST en la ruta `/eur-ace-criteria` que permita agregar nuevos criterios EUR-ACE.
2. La solicitud debe incluir un cuerpo en formato JSON que contenga al menos los siguientes campos:
   - `codigo`: un código único para el criterio EUR-ACE.
   - `nombre`: el nombre del criterio EUR-ACE.
   - `descripcion`: una descripción del criterio EUR-ACE.
3. El backend debe validar que el campo `codigo` sea único. Si se intenta agregar un criterio con un código ya existente, el endpoint debe devolver un código de estado HTTP 400 y un mensaje de error claro.
4. Si el criterio se crea correctamente, el endpoint debe devolver un código de estado HTTP 201 y el objeto del criterio creado.
5. Se debe manejar adecuadamente cualquier error que pueda ocurrir durante la ejecución del endpoint, devolviendo un mensaje de error claro y un código de estado HTTP apropiado.

- [7798] [BE] Validar unicidad del código del criterio EUR-ACE en backend
