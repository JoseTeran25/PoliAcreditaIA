# HU7773 - Crear Resultado de Aprendizaje

## Descripción
Como usuario, quiero poder crear Resultados de Aprendizaje (RA) a través de un endpoint, para poder agregar nuevos resultados al sistema de manera eficiente.

## Criterios de Aceptación
1. Se debe crear un endpoint POST en la ruta `/learning-outcomes` que permita agregar nuevos Resultados de Aprendizaje.
2. La solicitud debe incluir un cuerpo en formato JSON que contenga al menos los siguientes campos:
   - `codigo`: un código único para identificar el Resultado de Aprendizaje.
   - `nombre`: el nombre del Resultado de Aprendizaje.
   - `descripcion`: una descripción del Resultado de Aprendizaje.
   - `tipo`: el tipo de Resultado de Aprendizaje.
3. El backend debe validar que el código del RA sea único por tipo. Si se intenta crear un RA con un código que ya existe para ese tipo, el endpoint debe devolver un código de estado HTTP 400 y un mensaje de error claro.
4. Si el RA se crea correctamente, el endpoint debe devolver un código de estado HTTP 201 y un objeto que confirme la creación del RA, incluyendo su ID.
5. Se debe manejar adecuadamente cualquier error que pueda ocurrir durante la ejecución del endpoint, devolviendo un mensaje de error claro y un código de estado HTTP apropiado.

## Tareas
- [7782] [BE] Crear endpoint POST /learning-outcomes para agregar RAs
- [7783] [BE] Validar unicidad del código del RA por tipo en backend
