# HU8084 — Vincular RAA con RA

## Descripción
Como usuario, quiero poder vincular Resultados de Aprendizaje Asociados (RAA) con Resultados de Aprendizaje (RA), de manera que se pueda establecer una relación clara entre ellos y definir el nivel de aporte de cada RA a los RAA.

## Criterios de Aceptación
1. El sistema debe permitir la creación de relaciones entre RAA y RA, especificando el nivel de aporte de cada RA a su respectivo RAA.
2. La validación debe asegurarse de que las relaciones creadas sean coherentes y que los datos ingresados sean correctos.
3. El sistema debe almacenar correctamente las relaciones en la base de datos y permitir su consulta posterior.
4. El sistema debe manejar adecuadamente los casos en los que se intenten crear relaciones inválidas.

## Tareas (Backend)
- [ ] Crear o extender modelos de datos para almacenar las relaciones entre RAA y RA, incluyendo el nivel de aporte (Tarea ID: 8160).
- [ ] Crear un endpoint API `POST /api/learning-outcomes-associations` para registrar la relación RAA-RA y el nivel de aporte (Tarea ID: 8164).
- [ ] Implementar la lógica de validación de la relación RAA-RA y el nivel de aporte, asegurando que se cumplan todas las reglas de negocio (Tarea ID: 8165).
- [ ] Implementar pruebas unitarias para asegurar que la vinculación de RAA con RA funciona correctamente y que se manejan los errores adecuadamente.
- [ ] Documentar el nuevo endpoint y su uso en la API.

## Tareas (Frontend)
- [ ] Crear una interfaz que permita al usuario seleccionar un RAA y uno o más RAs para vincularlos, así como definir el nivel de aporte.
- [ ] Implementar la lógica para enviar la solicitud al backend con los datos de la relación RAA-RA.
- [ ] Mostrar un mensaje de éxito o error al usuario tras intentar crear la relación.
- [ ] Permitir la visualización de las relaciones existentes entre RAA y RA en la interfaz de usuario.
