# HU7753 - Registrar Justificación de Mapeo Directo

## Descripción
Como usuario, quiero poder registrar una justificación para un mapeo directo específico, para documentar las razones detrás de la relación entre los elementos mapeados.

## Criterios de Aceptación
1. Se debe crear un endpoint PUT/PATCH en la ruta `/mappings/{id}/justification` que permita guardar la justificación de un mapeo individual.
2. El endpoint debe aceptar un parámetro de ruta `{id}` que corresponda al identificador del mapeo para el cual se está registrando la justificación.
3. La solicitud debe incluir un cuerpo en formato JSON que contenga al menos el siguiente campo:
   - `justification`: un texto que explique la justificación del mapeo.
4. El endpoint debe validar que el campo `justification` no esté vacío y devolver un error adecuado si no se cumple esta condición.
5. Si la justificación se guarda correctamente, el endpoint debe devolver un código de estado HTTP 200 y un mensaje de éxito.
6. En caso de que el mapeo no exista, el endpoint debe devolver un código de estado HTTP 404 con un mensaje de error claro.

## Tareas
- [7785] [BE] Crear/adaptar endpoint PUT/PATCH /mappings/{id}/justification para guardar justificación de mapeo individual
