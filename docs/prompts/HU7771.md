# HU7771 - Crear Relación de Mapeo OPP vs RA (Asistente)

## Descripción
Como usuario, quiero poder crear relaciones de mapeo entre Objetivos de Programa (OPP) y Resultados de Aprendizaje (RA) en lote, para facilitar la gestión y justificación de estas relaciones en el sistema.

## Criterios de Aceptación
1. Se debe crear un endpoint POST en la ruta `/mappings/opp-ra/batch` que permita guardar múltiples relaciones de mapeo entre OPPs y RAs.
2. La solicitud debe incluir un cuerpo en formato JSON que contenga un array de objetos, donde cada objeto represente una relación de mapeo y contenga al menos los siguientes campos:
   - `oppId`: el identificador del Objetivo de Programa.
   - `raId`: el identificador del Resultado de Aprendizaje.
   - `justificacion`: una justificación para la relación de mapeo.
3. El backend debe implementar validaciones para evitar duplicados en las relaciones de mapeo. Si se intenta crear una relación que ya existe, el endpoint debe devolver un código de estado HTTP 400 y un mensaje de error claro.
4. Si las relaciones se crean correctamente, el endpoint debe devolver un código de estado HTTP 201 y un objeto que confirme la creación de las relaciones.
5. Se deben adaptar los endpoints existentes para listar OPPs y RAs para que sean compatibles con el uso en el asistente, incluyendo funcionalidades de búsqueda, paginación y filtrado.
6. Se debe manejar adecuadamente cualquier error que pueda ocurrir durante la ejecución del endpoint, devolviendo un mensaje de error claro y un código de estado HTTP apropiado.

- [7854] [BE] Adaptar endpoints de listar OPPs y RAs para uso en asistente (búsqueda/paginación/filtro)
