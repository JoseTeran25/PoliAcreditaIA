# HU8082 — Paginar Resultados de Aprendizaje (RA) para selección

## Descripción
Como usuario, quiero poder paginar los Resultados de Aprendizaje (RA) listados, de manera que pueda navegar fácilmente a través de grandes conjuntos de datos y seleccionar los RAs que necesito.

## Criterios de Aceptación
1. El sistema debe permitir la paginación de los resultados de aprendizaje, mostrando un número limitado de RAs por página.
2. La respuesta del endpoint debe incluir información sobre la paginación, como el número total de RAs, el número de páginas y los RAs de la página actual.
3. El usuario debe poder navegar entre las páginas de resultados de manera sencilla.
4. El sistema debe manejar correctamente los casos en los que no existan RAs o cuando se solicite una página que no tenga resultados.

## Tareas (Backend)
- [ ] Extender la lógica de paginación en el endpoint de listado de RAs para permitir la consulta de un número específico de RAs por página (Tarea ID: 8158).
- [ ] Implementar pruebas unitarias para asegurar que la paginación funciona correctamente y devuelve los resultados esperados.
- [ ] Documentar el nuevo comportamiento del endpoint y sus parámetros de paginación en la API.

## Tareas (Frontend)
- [ ] Implementar controles de paginación en la interfaz de usuario que permitan al usuario navegar entre las páginas de RAs.
- [ ] Modificar la lógica para enviar solicitudes al backend que incluyan parámetros de paginación.
- [ ] Mostrar la información de paginación, como el número total de RAs y el número de páginas, en la interfaz de usuario.
- [ ] Manejar el caso en que no se encuentren resultados y mostrar un mensaje adecuado al usuario.
